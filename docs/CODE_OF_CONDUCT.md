# Code of Conduct

## Purpose

VoidAccess exists to help security defenders investigate threats, understand adversary TTPs, and protect systems. That's the only purpose of this project. If you're here to learn to attack people, cause harm, or find targets, you have the wrong project.

## What's Expected

Stay on topic. This community is for security research — improving the tool, discussing methodology, sharing findings relevant to dark web OSINT and threat intelligence.

Disagree on technical grounds. Say why something doesn't work, show evidence, link to something that does. Don't make it personal.

If someone asks a basic question and you can help, help them. If you can't, let someone else do it.

Credit people. If you use someone's code, idea, or research, say where it came from.

## What Isn't Acceptable

Don't post malicious content, actual exploits, or illegal material in issues, pull requests, or discussions. This isn't a gray area.

Don't use this project's spaces to plan or discuss offensive operations, target real individuals, or do anything outside defensive security research.

Harassment, discrimination, and personal attacks get you removed. No long list of prohibited categories — use common sense.

Don't manipulate community members into misusing the tool. That's not a gray area either.

## Enforcement

Contact Katriel Moses at katriel.moses@gmail.com to report violations.

First violations get a warning. Serious violations or repeated minor ones mean removal from this community. There's no appeals process for anything involving illegal content.

Reports are kept private.

## Scope

This code applies to GitHub issues, pull requests, discussions, and anywhere the project is officially represented. This code of conduct also applies to any future community spaces such as Discord servers or forums associated with this project.
